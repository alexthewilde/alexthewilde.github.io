Behavior Driven Development

is a workflow for writing software "outside-in"

About BDD

Behavior driven development is an evolution of classic test driven development. In this process, before you think about test or implementation code, you start with describing how your software should behave in certain scenarios using plain language that everyone understands.

This extra step of specifying software behavior makes writing test code afterwards much easier and keeps an up-to-date documentation of your working software.

BDD with Drupal talk at DrupalCamp Essen 2012

Drupal Modules

Behat download

Behat is a PHP clone of Cucumber , the popular Ruby BDD tool. The Behat integration Drupal module makes it possible to write scenarios for a Drupal site using SimpleTest API functions within step definitions. learn more

Spec download

The Spec module takes a different approach than Behat and keeps all specs and test code in plain SimpleTest files. learn more

SimpleTest Fixture download

This little helper module speeds up test setup times and saves you from writing fixture boilerplate code, i.e. code to set up the required content and configuration required by a test.
